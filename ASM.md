### Hidden registers (inacessible by instructions directly except where needed for the instruction to function):
    pointer registers:
        Pl (32 bit) (lower 32 bits of the address pointer)
        Ph (16 bit) (upper 16 bits of the address pointer)
        
        interrupt index (48 bit) (set by the INTE instruction)
    
        program counter (48 bit) (counts your program)
    misc:
        carry flag (1 bit) (generated by ADD,AEC,SUB,and SWC instructions)
        stack pointer (16 bit) (increments by one, bit shifted to access 32 bit chunks)
        function pointer (16 bit) (increments by one, bit shifted to access 64 bit chunks)
        
    conditional registers: (set via a CMP instruction and used in any conditional branch)
        EQ (1 bit)
        less (1 bit)
        low (1 bit)

### Quirks:
    any instruction with a greater than 32 "immediate" without the immediate bit set raises a -1 value interrupt 
    (or unsigned, 4294967295)

    immediates are always sign extended

    16 bit immediates are sign extended to 48 bit for branches


### Instructions (in opcode order)

    ALU 		#all alu operations work on two registers and return to a third register.
    add		#add 
    awc		#add with carry flag
    sub		#subtract
    swc		#subtract with carry flag
    nand	#bitwise NAND
    and		#bitwise AND
    rol		#rotate left
    ror		#rotate right
    
    COND		#branches and jumps
    jum		#jump always. 
    bra		#branch always. takes one register argument as the amount to branch (signed).
    beq		#branch if a and b are equal. takes one register argument as the amount to branch (signed).
    bne		#branch if a and b are not equal. takes one register argument as the amount to branch (signed).
    bg		#branch if a is greater than b. takes one register argument as the amount to branch (signed).
    bl		#branch if a is less than b. takes one register argument as the amount to branch (signed).
    ba		#branch if a is above b. takes one register argument as the amount to branch (signed).
    bb		#branch if a is below b. takes one register argument as the amount to branch (signed).
    
    EXTRA 	#various extras, mainly for the OS
    ioio	#sets the address to jump to on interrupt on io inputs.	
    call	#force an interrupt.
    ret		#return from an interrupt/jump.
    push	#push a register to the stack.
    pull	#pull a register from the stack.
    cmp		#compare two registers
    llp		#load lower pointer.
    lup		#load upper pointer.
    
    SPECIAL
    lpc		#load lower 32 bits of program counter.
    lupc 	#load upper 16 bits of program counter.
    load	#load from a position denoted by the first two arguments, into the third register argument.
    store 	#store from the third register arguments at the location denoted by the first two.
    and		#bitwise and two values.
    key		#load the last interrupt value
    nand	#bitwise nand two values.
    lui		#load upper immediate; load the higher 16 bits of a register with an immediate value.


### instructions are formatted like this: (all arguments are optional except opcode)

    opcode rs1,rs2,rd
    or
    opcode rs1,immediate,rd


some instructions don't use these though.
    
    branches only take a register/immediate amount to branch, as well as LLP, LUP, and STORE
    
    IOIO, CALL,and RET take zero arguments
    
    LPC,LUPC, PULL, STORE, and KEY all take one register destination argument
    
    and lastly CMP takes two register arguments or a register and an immediate.
